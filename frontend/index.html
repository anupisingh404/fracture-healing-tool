Platform Overview

Everything a Clinician Needs, Instantly

From raw biomarker values to a complete clinical narrative — in seconds.

📊

Biomarker Analysis

Tracks BSAP, ALP, and P1NP across Day 1, Week 3, and Week 6. Computes trend deltas and flags abnormal patterns automatically.

🤖

5-Model ML Ensemble

Random Forest, XGBoost, Logistic Regression, SVM, and Gradient Boosting run in parallel. The best model's result is highlighted; all scores are displayed.

📡

RAG-Powered Case Retrieval

ChromaDB vector store finds the 3 most similar historical patients using semantic similarity on biomarker summaries.

🌐

Tavily Literature Search

Retrieves relevant fracture healing research from the web in real time, grounding the AI explanation in current medical evidence.

💬

GPT-4o Clinical Narrative

OpenAI GPT-4o synthesises ML results, biomarker trends, similar cases, and literature into a structured 3–4 paragraph clinical explanation.

🔗

MCP Integration

All 4 prediction tools are exposed via Model Context Protocol — register the server in Claude Desktop for agentic clinical workflows.

The Problem

Current Monitoring Falls Short

Fracture healing is primarily assessed through X-rays and clinical symptoms — methods that detect problems after the optimal intervention window has closed.

❌
X-rays show healing changes late

Radiological confirmation of union typically takes 6–12 weeks

❌
No integrated biomarker system

BSAP, ALP, and P1NP data are collected but never synthesised for prognosis

❌
Non-union identified too late

5–10% of fractures develop non-union; current tools catch it at 3–6 months

Our Solution

Early Prediction at Week 3

FractureAI uses biochemical markers collected routinely in orthopaedic care to generate a healing prediction weeks before radiological signs appear.

✅
Predict at Week 3, intervene at Week 4

Actionable risk stratification while intervention is still effective

✅
Integrates all available data

Demographics, 15 biomarker readings, and callus measurements combined

✅
Explainable AI output

Not a black box — every prediction comes with a clinical narrative and evidence links

Workflow

Six Steps from Input to Insight

The platform automates a complex clinical pipeline in under 5 seconds.

1

Enter Patient Data

Input demographics, biomarker values at three time points, and callus measurements via the dashboard form.

2

Feature Engineering

The backend computes 32-dimensional feature vectors including trend deltas, ratios, and callus growth rate.

3

5-Model Prediction

Five ML classifiers run in parallel. XGBoost leads; all scores are shown for transparency.

4

Similar Case Retrieval

ChromaDB finds 3 historically similar patients using MiniLM embeddings and cosine similarity.

5

Literature Search

Tavily queries medical databases for current evidence on the patient's fracture type and biomarker pattern.

6

Clinical Report

GPT-4o synthesises everything into a structured clinical narrative with risk flags and recommendations.

Technology

Built on Production-Grade Components

FastAPI

High-performance async Python API. Serves both REST endpoints and static frontend.

Chart.js

Interactive biomarker trend charts rendered client-side with no bundler required.
